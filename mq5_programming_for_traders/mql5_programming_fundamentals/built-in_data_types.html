内置数据类型 ​

数据类型是一个我们在日常生活中常常会使用到的基本概念，甚至都没有意识到它的存在。它是基于我们所交换信息的含义以及对这些信息可允许的处理程序而隐含存在的。例如，在管理我们的家庭资产时，我们会对代表收入和支出的数字进行加减法运算。在这里，“数字”描述了一种类型，对于这种类型，我们完全了解它可能的取值以及对其进行的算术运算。在交易场景中，MetaTrader 5 里的当前账户余额也是类似的值；因此，MQL5 提供了一种创建和操作数字的机制。

与数字不同，文本信息，比如交易工具的名称，遵循其他规则。在这里，我们可以用字母组成单词，或者用单词组成句子，但不可能计算几行文本的累计总和或算术平均值。所以，“行”或“字符串”是另一种数据类型，而不是数值类型。

除了每种类型的用途和一组典型的有意义的操作之外，还有另一个使不同类型相互区别的重要因素，那就是它们的大小。例如，一年中的周数不会超过 52，而从年初开始已经过去的秒数则是一个非常大的数字。因此，为了在计算机内存中高效地存储和处理这些不同的值，可以划分出不同大小的存储段。这让我们明白，实际上，“数字”这个笼统的概念可能隐藏着不同的类型。

MQL5 允许使用一些数字类型，这些类型不仅在所分配的内存单元大小上有所不同，而且在一些附加特性上也有差异。特别是，有些数字可以取负值，比如以点为单位的浮动盈利，而另一些数字则不能，比如账号。此外，有些值不能有小数部分，因此，与随意的“带小数点的数字”相比，用更严格的“整数”类型来表示它们会更节省成本。例如，账户余额或交易工具的价格通常是带小数点的值。同时，历史订单数量或者账号总是整数。

MQL5 支持一组通用类型，与绝大多数编程语言中可用的类型类似。这组类型包括整数类型（不同大小）、两种不同精度的实数类型（带小数点）、字符串、单个字符，以及只有两种可能值（真和假）的逻辑类型。此外，MQL5 还提供了其特有的用于处理时间和颜色的类型。

为了内容的完整性，需要指出的是，MQL5 允许扩展类型集合，在代码中声明应用类型，即结构体、类和其他面向对象编程（OOP）中常见的实体；但我们将在后面讨论它们。

由于存储值的单元大小是一个重要的类型属性，让我们来谈谈内存相关的方法。

计算机内存的最小单位是字节。换句话说，字节是程序可以为单个值分配的最小单元大小。一个字节由 8 个更小的“粒子”（位）组成，每个位可以处于两种状态：开启（1）或关闭（0）。所有现代计算机在底层都使用这样的位，因为这种二进制的信息表示形式便于在硬件中实现（在随机存取存储器、处理器中，或者在通过网络电缆或 WiFi 传输数据时）。

对不同类型值的处理是通过对内存单元中位状态的不同解释来实现的。这由编译器来处理。程序员通常不会深入到处理位的层面；不过，该语言提供了相应的工具（见“位运算”）。

在 MQL5 中有特殊的保留字来描述数据类型。我们在第 1 部分已经知道了其中一些，比如 void、int 和 string。下面给出了完整的类型列表，每个类型都有简要说明和以字节为单位的大小。

根据它们的用途，可以有条件地将它们分为数值型和字符编码型数据（在相关列中标记），以及其他专门的类型，如字符串、逻辑（或布尔）类型、日期/时间和颜色。void 类型比较特殊，表示根本没有值。除了标量类型，MQL5 还提供了用于处理复数、矩阵和向量的对象类型：complex、vector 和 matrix。这些类型用于解决线性代数、数学建模、机器学习和其他领域的各种问题。我们将在本书的第 4 部分详细研究它们。

根据其大小，不同的数值类型可以存储不同的取值范围。除了上述情况，相同大小的整数和浮点数的取值范围可能会有很大差异，因为它们使用了不同的内部表示方式。所有这些复杂的内容将在处理特定类型的部分进行讨论。

程序员可以根据预期的值、效率方面的考虑或出于节省资源的原因自由选择数值类型。特别是，较小的类型大小允许在内存中存储更多该类型的值，而且整数的处理速度比浮点数快。

请注意，数值型和字符编码型类型在一定程度上有交叉。这是因为字符在内存中是作为整数存储的，即在相关的字符表中的一个代码：对于单字节字符是 ANSI 码，对于双字节字符是 Unicode 码。ANSI 是一个以机构（美国国家标准协会）命名的标准，而 Unicode，正如你所猜测的，意思是通用代码（字符集）。在 MQL5 中使用 Unicode 字符来创建字符串（string 类型）。在将程序与外部数据源集成时，通常需要单字节字符，比如来自互联网的数据源。

如前所述，数值类型可以分为整数和浮点数。让我们更详细地研究一下它们。

整数 ​

整数类型用于存储没有小数点的数字。如果某个值的实际应用意义不包含小数部分，就应该选择整数类型。例如，图表上的柱线数量或未平仓头寸的数量总是整数。

MQL5 允许分别使用关键字 char、short、int 和 long 来选择大小为 1 到 8 字节的整数类型。它们都是有符号类型，也就是说，它们既可以包含正值，也可以包含负值。如果有必要，相同大小的整数类型可以声明为无符号类型（它们的名称以表示“无符号”的“u”开头）：uchar、ushort、uint 和 ulong。

没有必要记住每个整数的上述极限值。在 MQL5 中有许多预定义的命名常量，在代码中可以使用它们来代替“魔法数字”，其中包括最小/最大整数。在处理预处理器的部分会介绍这种技术。在这里，我们仅列出相关的命名常量：CHAR_MIN、CHAR_MAX、UCHAR_MAX、SHORT_MIN、SHORT_MAX、USHORT_MAX、INT_MIN、INT_MAX、UINT_MAX、LONG_MIN、LONG_MAX 和 ULONG_MAX。

让我们解释一下这些值是如何得到的。这需要回到位和字节的概念。

一个字节中 8 位的不同状态（开启和关闭）的所有可能组合的数量是 256。这就产生了可以存储在一个字节中的 0 到 255 的取值范围。然而，对它们的解释取决于为这个字节分配的类型。编译器会根据程序员的声明来确保不同的解释方式。

一个字节中的低位（最右边）位表示 1，第二位表示 2，第三位表示 4，以此类推，直到高位位表示 128。很明显，这些数字等于 2 的幂次方，幂次等于位的编号（编号从 0 开始）。这就是使用二进制系统的结果。

位高位低位
编号76
值12864

当所有位都被置位时，就会得到 2 的所有幂次的和，即 255 是一个字节的最高值。如果所有位都被复位，我们就得到 0。如果低位位被置位，这个数字就是奇数。

在对有符号数字进行编码时，高位位用于标记负值。因此，对于单字节整数的正值范围，127 成为最高值。对于负值，有 128 种可能的组合，即最低值是 -128。当一个字节中的所有位都被置位时，它被解释为 -1。如果在这样的数字中低位位被复位，我们就会得到 -2，以此类推。如果只有高位位（符号位）被置位，而其他所有位都被复位，我们就得到 -128。

这种看起来可能不合理的编码方式被称为“补码”。它允许在硬件层面统一对有符号数和无符号数的计算。此外，它避免了如果正负区域以相同方式编码会出现的值的丢失问题：否则，我们对于 0 就会有两个值，即正 0 和负 0。而且，这会带来歧义。

具有更多字节（即 2 字节、4 字节或 8 字节）的数字也有类似的连续位编号和相应值的递增规律。在所有情况下，判断一个数字为负数的标准是高位字节的高位位被置位。

因此，我们可以使用一个字节来存储 0 到 255 范围内的无符号整数（uchar，即 unsigned character 的缩写）。我们也可以将有符号整数写入这个字节（为此我们将其类型描述为 char）。在这种情况下，编译器会将 256 种可用的组合数量平均分配给正值和负值，将其映射到 -128 到 127 的区域（第 256 个值是 0）。很明显，对于有符号和无符号字节，0 到 127 的值在位级别上的编码是相同的。然而，从 128 开始的大绝对值会变成负数（根据上面插入部分描述的方案）。这种“转换”只发生在读取存储的值或对其执行任何操作的时刻，而内部数据表示（位的状态）是相同的。

我们将在处理类型转换的部分更详细地讨论这个问题。

与单字节整数类似，很容易计算出 2 字节的位组合数量是 65536。因此，形成了有符号和无符号的双字节整数 short 和 ushort 的取值范围。其他类型由于字节大小增加，可以存储更大的值。

请注意，使用相同大小的无符号类型可以使最高正值翻倍。对于存储可能非常大且不会出现负值的数量，这可能是必要的。例如，MetaTrader 5 中的订单号是 ulong 类型的值。

我们在第 1 部分已经遇到过整数描述的示例。特别是，在那里定义了 uint 类型的输入参数 GreetingHour：

c
input uint GreetingHour = 0;

除了使变量在 MQL 程序的参数列表中可见的额外关键字 input 之外，其他组件，即类型、名称以及在“=”号后面的可选初始化，是所有变量都具有的基本要素。

变量描述的语法将在“变量”部分详细讨论。目前，请注意记录整数类型常量的方法。在描述变量时，常量可以指定为默认值（在上面的示例中是 0）。此外，常量可以用于表达式中，例如，在公式事件中。

应该提醒的是，插入到源代码中的任何类型的常量都称为字面量（从字面上看：“逐字逐句”）。它们的名称源于这样一个事实，即它们“按原样”引入到程序中，并在描述的位置立即使用。与语言的许多其他元素（特别是变量）不同，字面量没有名称，并且不能从程序的其他位置引用。

对于负数，需要在数字前面提供负号“-”；然而，对于正数，正号“+”可以省略，即 +100 和 100 这两种形式是相同的。

应该注意的是，数值通常在源代码中以我们习惯的十进制表示法记录。然而，MQL5 允许使用另一种表示法，即十六进制。这对于处理位级信息很方便（见“位运算”）。

在十进制常量的所有数位中允许使用 0 到 9 的数字，而对于十六进制常量，除了数字之外，还额外使用从 A 到 F 或从 a 到 f 的拉丁字母符号（也就是说，大小写无关）。十六进制数字 A 对应十进制表示中的数字 10，B 对应 11，C 对应 12，依此类推，直到 F 等于 15。

十六进制常量的一个显著特点是它以前缀 0x 或 0X 开头，后面跟着数字的有效数位。例如，数字 1 在十六进制系统中记录为 0x1，而 16 记录为 0x10（需要一个额外的高位数位，因为 16 大于 15，即 0xF）。十进制的 255 变成 0xFF。

让我们给出更多示例，说明在描述变量时使用整数类型的各种情况（附在脚本 MQL5/Scripts/MQL5Book/p2/TypeInt.mq5 中）：

c
void OnStart()
{
  int x = -10;          // 可以，有符号整数 x = -10
  uint y = -1;          // 可以，但无符号整数 y = 4294967295
  int z = 1.23;         // 警告：常量值截断，z = 1
  short h = 0x1000;     // 可以，h 在十进制中为 4096
  long p = 10000000000; // 可以
  int w = 10000000000;  // 警告，截断...，w = 1410065408
}

变量 x 被正确初始化为允许的负值 -10。

变量 y 是无符号的。因此，尝试在其中记录一个负值会产生一个有趣的效果。数字 -1 在位上有一个表示形式，程序会根据无符号类型 uint 对其进行解释。因此，得到数字 4294967295（实际上它等于 UINT_MAX）。

变量 z 被赋予了浮点数 1.23（我们将在下一部分讨论浮点数），编译器会警告小数部分被截断。结果，整数 1 被存储到变量中。

变量 h 成功地被十六进制形式的常量（0x1000 = 4096）初始化。

大值 10000000000 被记录在变量 p 和 w 中，前者是长整数类型（long），处理成功，而后者是普通类型（int），因此会引发编译器警告。由于该常量超过了 int 的最大值，编译器会截断多余的高位数位（位），实际上，1410065408 被存储到了 w 中。

这种行为是类型转换可能出现的潜在不良情况之一，这可能是程序员有意或无意造成的。在后一种情况下，这可能会导致潜在的错误。显然，在这个特定示例中，故意选择了错误的值来演示警告。在实际程序中，程序试图存储在整数变量中的值并不总是那么明显。因此，你应该非常仔细地查看编译器警告，并尝试通过更改类型或显式指定所需的类型转换来消除它们。这将在处理“类型转换”的部分进行讨论。

对于整数类型，定义了算术运算、位运算和其他类型的运算（见“表达式”章节）。

浮点数 ​

在日常生活中，我们使用带小数点的数，即实数，的频率与使用整数的频率一样高。“实数”这个名称本身就表明，使用这类数字，你可以表达现实世界中一些实实在在的东西，比如重量、长度、体温等，也就是所有不能用整数个单位来度量，而是“多一点”的事物。

我们在交易中也经常使用实数。例如，它们用于表示交易订单中的标的价格或交易量（通常允许有整手合约的小数部分）。

MQL5 提供了两种实数类型：普通精度的 float 类型和双倍精度的 double 类型。

在源代码中，float 和 double 类型的常量值通常记录为整数部分和小数部分（各为一串数字），由字符“.”分隔，例如 1.23 或 -789.01。可以没有整数部分或小数部分（但不能两者都没有），但小数点是必须的。例如，.123 表示 0.123，而 123. 表示 123.0。单纯的 123 会创建一个整数类型的常量。

然而，还有另一种记录实型常量的形式，即指数形式。在这种形式中，整数部分和小数部分后面跟着 E 或 e（大小写无关）以及一个表示幂的整数，将 10 提升到这个幂次就得到一个额外的因数。例如，以下表示形式以指数形式显示了同一个数字 0.57：

.0057e20.057e1.057e157e-2

在记录实型常量时，默认情况下，后者被定义为 double 类型（它们占用 8 个字节）。要设置为 float 类型，应该在常量的右边添加后缀 F（或 f）。

float 和 double 类型在大小、取值范围和数字表示精度上有所不同。所有这些都显示在下面的表格中。

类型大小（字节）最小值最大值精度（有效数字位数）
float4±1.18 * 10⁻³⁸±3.4 * 10³⁸6 - 9，通常为 7
double8±2.23 * 10⁻³⁰⁸±1.80 * 10³⁰⁸15 - 18，通常为 16

取值范围以绝对值表示：最小值和最大值确定了在正负两个区域内允许值的幅度。与整数类型类似，对于这些极限值也有内置的命名常量：FLT_MIN、FLT_MAX、DBL_MIN、DBL_MAX。

请注意，实数总是有符号的，也就是说，不存在无符号的实数类型。

精度是指相关类型的实数能够无失真存储的有效数字（十进制数字）的数量。

实际上，实数类型的数字不像整数类型的数字那样精确。这是为它们的通用性和更广泛的潜在取值范围所付出的代价。例如，如果一个无符号 4 字节整数（uint）的最大值是 4294967295，即大约 40 亿，或 4.29*10⁹，那么 4 字节实数（float）的最大值是 3.4 * 10³⁸，比前者高 29 个数量级。对于 8 字节类型，差异更加明显：ulong 可以存储 18446744073709551615（18.44*10¹⁸，或约 18 京），而 double 可以存储 1.80 * 10³⁰⁸，即比前者高 289 个数量级。关于精度的更多细节见下文。

尾数和指数 ​

实数在内存中（在为它们分配的字节中）的内部表示相当复杂。最高位用作负号的标志（我们在整数类型中也看到过这种情况）。所有其他位被分为两组。较大的一组包含数字的尾数，即有效数字（这里指的是二进制位，即位）。较小的一组存储幂（指数），将 10 提升到这个幂次，再与尾数相乘就得到存储的数字。具体来说，对于 float 类型，尾数的大小是 24 位（FLT_MANT_DIG），而对于 double 类型，尾数大小是 53 位（DBL_MANT_DIG）。按照常规的十进制数位（数字）来计算，我们会得到与上面表格中相同的精度：float 的最低有效数字数量是 6（FLT_DIG），而 double 的是 15（DBL_DIG）。然而，根据具体的数字，可能会有“幸运”的位组合，对应着更多的十进制数字。对于 float 和 double，指数部分的大小分别是 8 位和 11 位。

由于有指数，实数获得了大得多的取值范围。同时，随着指数的增加，尾数的低位数字的“权重”也会增加。这意味着在计算机内存中可以表示的两个相邻实数有很大的差异。例如，对于数字 1.0，在 float 类型下，最低位的“权重”是 1.192092896e—07（FLT_EPSILON），在 double 类型下是 2.2204460492503131e-016（DBL_EPSILON）。换句话说，如果一个数小于 1.192092896e—07，那么 1.0 就与它附近的任何数无法区分。这可能看起来不是很重要或者“没什么大不了的”，但对于较大的数字，这个不确定区域会变得更大。如果你用 float 存储一个大约 10 亿（1*10⁹）的数字，最后两位数字将无法安全地存储或从内存中恢复（见下面的代码示例）。然而，基本上，问题不在于数字的绝对值，而在于其中能够无损失地召回的最大数字位数。同样地，我们可以尝试将一个表示为 1234.56789（在结构上很像金融工具的价格）的数字存储在 float 类型中；由于其内部表示缺乏精度，它的最后两位数字将会“浮动”。

对于 double 类型，类似的情况会在数字大得多（或者有效数字数量多得多）时出现，但在实践中仍然是可能发生且经常发生的。在处理非常大或非常小的实数时，你应该考虑到这一点，并在编写程序时对潜在的精度损失进行额外的检查。特别是，你应该以一种特殊的方式将实数与零进行比较。我们将在比较运算符部分处理这个问题。

细心的读者可能会觉得上面给出的尾数和指数的大小是错误的。让我们以 float 类型为例来解释一下。它存储在大小为 4 字节的内存单元中，即占用 32 位。同时，尾数（24 位）和指数（8 位）的大小之和已经是 32 位了。那么符号位在哪里呢？问题在于，信息技术专业人员将尾数存储为“规范化”形式。如果我们先考虑一个普通十进制数的指数表示形式，就更容易理解这是什么意思了。比如说，数字 123.0 可以表示为 1.23E2、12.3E1 或 0.123E3。一种表示被认为是规范化形式，即小数点前面只有一个有效数字（即不是零）。对于这个数字，规范化形式是 1.23E2。根据定义，在十进制表示中，数字 1 到 9 被认为是有效数字。现在我们顺利过渡到二进制表示。在二进制中只有一个有效数字，即 1。看来二进制表示中的规范化形式总是以 1 开头，并且这个 1 可以省略（无需在它上面花费内存）。通过这种方式，尾数中可以节省一位。实际上，尾数包含 23 位（还有一个隐含的高位 1，在从内存中重构和检索数字时会自动添加）。尾数减少 1 位就为符号位腾出了空间。

通常，在应该使用浮点类型的地方，我们选择 double 类型，因为它更精确。float 类型仅用于节省内存，例如在处理非常大的数据数组时。

在脚本 MQL5/Scripts/MQL5Book/p2/TypeFloat.mq5 中展示了一些使用实型常量的示例。

c
void OnStart()
{
  double a0 = 123;      // 可以，a0 = 123.0
  double a1 = 123.0;    // 可以，a1 = 123.0
  double a2 = 0.123E3;  // 可以，a2 = 123.0
  double a3 = 12300E-2; // 可以，a3 = 123.0
  double b = -.75;      // 可以，b = -0.75
  double q = LONG_MAX;  // 警告：截断，q = 9.223372036854776e+18
                        //               LONG_MAX = 9223372036854775807
  double d = 9007199254740992; // 可以，double 中能稳定存储的最大长整数
  double z = 0.12345678901234567890123456789; // 可以，但会截断
                           // 为 16 位：z = 0.1234567890123457
  double y1 = 1234.56789;  // 可以，y1 = 1234.56789
  double y2 = 1234.56789f; // 精度损失，y2 = 1234.56787109375
  float m = 1000000000.0;  // 可以，按原样存储
  float n =  999999975.0;  // 警告：截断，n = 1000000000.0
}

变量 a0、a1、a2 和 a3 包含相同的数字（123.0），只是以不同的方式书写。

在变量 b 的常量中，小数点前面的无意义的零被省略了。此外，这里展示了使用负号“-”记录负数的方法。

尝试在变量 q 中存储最大整数。此时，编译器会给出一个警告，因为 double 不能精确表示 LONG_MAX：得到的不是 9223372036854775807，而是 9223372036854776000。这明显表明，即使 double 的取值范围远远超过整数的取值范围，但这是通过丢失低位数字实现的。

作为对比，变量 d 的值给出了 double 类型能够无失真存储的最大整数。在整数序列中，如果我们用 double 来表示它们，在这个整数之后会有零星的跳跃。

变量 z 再次提醒我们关于最大有效数字数量（16）的限制——更长的常量将被截断。

变量 y1 和 y2 中以不同的格式（double 和 float）记录了同一个数字，这使得我们可以看到转换为 float 时的精度损失。

实际上，变量 m 和 n 将会相等，因为 999999975.0 在内部表示中被近似存储，并变成了 1000000000.0。

数值类型经常用于使用公式进行计算；为它们定义了广泛的操作集（见表达式部分）。

计算有时会导致不正确的结果，也就是说，这些结果不能表示为一个数字。例如，负数的平方根或零的对数是无法定义的。在这种情况下，实数类型可以存储一个特殊的值，名为 NaN（非数）。实际上，有几种不同类型的这样的值，例如，可以区分正无穷和负无穷。MQL5 提供了一个特殊的函数 MathIsValidNumber，用于检查 double 值是一个数字还是 NaN 值之一。

字符类型 ​

字符数据类型旨在存储特定的字符（字母），字符串就是由这些字符组成的（请参阅“字符串”）。MQL5 有 4 种字符类型：两种 1 字节大小的（char、uchar）和两种 2 字节大小的（short、ushort）。前缀为“u”的类型是无符号的。

实际上，字符类型是整数类型，因为它们存储的是相关字符表中字符的整数值代码：对于 char 类型，它是 ASCII 字符表（代码 0-127）；对于 uchar 类型，它是扩展 ASCII 码（代码 0-255）；对于 short/ushort 类型，它是 Unicode 表（无符号版本中最多有 65535 个字符）。如果你感兴趣的话，ASCII 是“美国信息交换标准代码”的缩写。

对于 MQL5 字符串，使用的是 2 字节的字符 ushort 类型。1 字节的 uchar 类型通常用于与外部程序集成，在根据应用协议（例如连接到加密货币平台时）传输以其他类型进行打包和解包的随机数据数组时会用到。

字符常量以单引号括起来的字母形式记录。不过，你也可以使用上面提到的整数表示法（请参阅“整数”）。同时，该整数必须在 1 字节或 2 字节格式的取值范围内。

此外，我们可以使用转义序列表示法。它们使用反斜杠（''）作为第一个字符，后面跟着一个预定义的控制字符和/或一个数字代码。MQL5 支持以下转义序列：

  • \n — 换行
  • \r — 回车
  • \t — 制表符
  • \ — 反斜杠
  • " — 双引号
  • ' — 单引号
  • \X 或 \x — 用于随后指定十六进制格式的数字代码的前缀
  • \0 — 用于随后指定八进制格式的数字代码的前缀

字符类型常量的基本使用方法在脚本 MQL5/Scripts/MQL5Book/p2/TypeChar.mq5 中给出。

c
void OnStart()
{
  char a1 = 'a';  // 没问题，a1 = 97，英文字母 'a' 的代码
  char a2 = 97;   // 没问题，a2 同样等于 'a'
  char b = '£';   // 警告：常量值被截断，b = -93
  uchar c = '£';  // 没问题，c = 163，英镑符号的代码
  short d = '£';  // 没问题
  short z = '\0';    // 没问题，0
  short t = '\t';    // 没问题，9
  short s1 = '\x5c'; // 没问题，反斜杠代码 92
  short s2 = '\\';   // 没问题，反斜杠本身，代码同样是 92
  short s3 = '\0134';// 没问题，八进制形式的反斜杠代码
}

变量 a1 和 a2 以两种不同的方式获取字符 'a'（英文字母）的值。

有一个将 '£' 记录到变量 b 中的尝试，但其代码 163 超出了 char 类型的范围（127）；因此它被“转换”为有符号的 -93（编译器会给出警告）。接下来的 uchar 类型变量（c）和 short 类型变量（d）将这个代码视为正常。

其他变量使用转义序列进行初始化。

字符可以使用与整数相同的操作进行处理（请参阅“表达式”）。

字符串类型 ​

字符串类型用于存储基于文本的信息，用关键字string表示。字符串是ushort字符的序列，支持完整的Unicode范围，包括多种国家文字。例如，金融工具的名称和交易订单中的注释都是字符串。

由于字符串的特殊性，其大小是一个可变的值，等于文本长度的两倍（字符数量乘以字符的“宽度”，即2字节）再加上一个额外的字符。这个额外的字符用于存储“终止零”（编码为0的字符），用于表示字符串的结束。此外，MQL5会使用一些空间来存储服务信息，即指向字符串在内存中起始位置的引用。

与C++不同，在MQL5中无法获取字符串或任何其他变量的地址。MQL5禁止直接访问内存。

字符串字面量在源代码中表示为用双引号括起来的字符序列。例如："EURUSD"或"$"。我们应该区分由单个字符组成的字符串（如"$"）和相同的单个字符（如'$'），它们是不同的数据类型。

空字符串表示为""。考虑到隐式的终止零，除了服务信息外，它占用2字节的内存。

如果需要在字符串中使用双引号字符，则必须在其前面加上反斜杠字符，将其转换为控制序列，例如"Press \"OK\""。

字符串初始化的示例见脚本MQL5/Scripts/MQL5Book/p2/TypeString.mq5。

c++
void OnStart()
{
   string h = "Hello";          // Hello
   string b = "Press \"OK\"";   // Press "OK"
   string z = "";               //
   string t = "New\nLine";      // New
                                // Line
   string n = "123";            // 123, 文本（不是整数值）
   string m = "very long message "
              "can be presented " 
              "by parts";
   // 等价于:
   // string m = "very long message can be presented by parts"; 
}

字符串"Hello"被赋值给变量h。

包含双引号的文本被写入变量b。

变量z被初始化为空字符串。这基本上等同于不进行初始化来描述z，但这里有一些细微的差别。在后续关于变量初始化的章节中，我们会了解到未初始化的字符串会有一个特殊的值NULL，这与""不同，如前所述，""会为终止零分配内存。这种差异会影响字符串比较运算符和其他一些运算符的执行。随着内容的深入，我们会涉及到所有这些方面。

变量t将得到一个文本，当使用Print函数将其打印到日志中或以其他方式显示时，会被分成两行。

记录在变量n中的字符串"123"虽然看起来像一个数字，但它并不是数字。MQL5中有一些函数可以将文本转换为数字，反之亦然（见“数据转换”部分）。此外，还有一组专门用于处理字符串的函数。

为了方便起见，长的字面量可以写成多个字符串，就像变量m那样。一般规则如下：编译器会将直到标记变量描述结束的分号之前的所有字面量合并。在这种格式中，关键是如果需要（例如，像上面的例子那样分隔消息中的单词），不要忘记在字符串的每个片段中添加中间的空格。

对于字符串，定义了求和（连接）操作，用字符+表示。我们将在处理表达式的章节中讨论它（见“算术运算”）。

可以像访问数组元素一样单独读取字符串中的字符：如果s是一个字符串，那么s[i]是其中第i个字符的代码，类型为ushort。

逻辑（布尔）类型 ​

逻辑类型用于存储仅有两种可能状态的特征，即“启用”/“禁用”。在许多程序（包括MetaTrader 5）的设置对话框中，其界面类似物就是各种选项：每个标志要么启用，要么禁用。通过检查这些特征的状态，程序执行逻辑能够实现分支，这也是该类型名称的由来。

在MQL5中，逻辑类型由bool关键字定义，占用1字节的内存。对于这种类型，保留了两个常量：true和false。此外，在实际情况中（程序员也经常利用这一点），bool类型可以作为整数和实数运算的结果，其中值0被解释为false，其他任何值都被解释为true。

同时，也支持将bool类型的值反向解释为数字：true被视为1，false被视为0。

逻辑类型变量的示例位于文件MQL5/Scripts/MQL5Book/p2/TypeBool.mq5中。

c++
void OnStart()
{
  bool t = true;          // true
  bool f = false;         // false
  bool x = 100;           // x = true
  bool y = 0;             // y = false
  int i = true;           // i = 1
  int j = false;          // j = 0
}

对于逻辑类型，提供了一组特殊的逻辑运算（请参阅“逻辑（布尔）运算和比较运算”）。

日期和时间 ​

MQL5 提供了一种特殊类型 datetime 用于存储时间数据。从其名称可以看出，datetime 类型的值既包含日期也包含时间。不过，如有需要，它们也可以仅包含日期或仅包含一天中的时间。

这种类型的值可在程序中用于监控事件，例如交易时间、新闻发布时间，或者在不良交易后临时禁用智能交易系统（EA）交易的超时时间。

datetime 类型在内存中占用 8 个字节。其数据的内部表示与 ulong 类型完全相同，因为内部存储的是自 1970 年 1 月 1 日以来经过的秒数。所支持的最大日期是 3000 年 12 月 31 日。

datetime 常量以单引号括起来的字符串字面量形式记录，并在前面加上字符 D。字符串内部划分为 6 个字段，日期和时间的所有组成部分的数字采用以下格式：

  • D'YYYY.MM.DD HH:mm:ss'
  • D'DD.MM.YYYY HH:mm:ss'

这里，YYYY 表示年份，MM 表示月份，DD 表示日期，HH 表示小时，mm 表示分钟，ss 表示秒。你可以省略日期或时间。也可以不指定秒，或者不指定分钟和秒。

对于允许的最大日期值，MQL5 提供了一个特殊常量 DATETIME_MAX，其整数值为 0x793406fff，对应于 D"3000.12.31 23:59:59"。

在文件 MQL5/Scripts/MQL5Book/p2/TypeDateTime.mq5 中展示了记录 datetime 类型值的示例：

c
void OnStart()
{
  // 警告：无效日期
  datetime blank = D'';           // blank = 编译日期
  datetime noday = D'15:45:00';   // noday = 编译日期 + 15:45
  datetime feb30 = D'2021.02.30'; // feb30 = 2021.03.02 00:00:00
  datetime mon22 = D'2021.22.01'; // mon22 = 2022.10.01 00:00:00
  // 正常
  datetime dt0 = 0;                      // 1970.01.01 00:00:00
  datetime all = D'2021.01.01 10:10:30'; // 2021.01.01 10:10:30
  datetime day = D'2025.12.12 12';       // 2025.12.12 12:00:00
}

前四个变量会引发编译器关于日期不正确的警告。对于 blank 变量，字面量完全为空。对于 noday 变量，没有指定日期。在这两种情况下，编译器会用编译日期替换常量中的日期。feb30 和 mon22 变量包含不正确的日期和月份数字。编译器会自动修正它们，将溢出部分转移到更高阶的字段（2 月 30 日变为 3 月 2 日，第 22 个月变为下一年的第 10 个月）。不过，始终建议消除这些警告。

变量 dt0 展示了用整数初始化 datetime 值的方式。

datetime 类型支持整数固有的一系列操作（见“表达式”）。例如，这允许在时间上加上预定义的秒数（得到未来的某个时刻），或者计算日期之间的差值。

颜色 ​

MQL5 有一种用于处理颜色的特殊类型。这使得对图形对象进行着色成为可能。

为了表示这种类型，使用了 color 关键字。对于 color 类型的值，会分配 4 个字节的内存。它的内部表示是一个无符号整数，包含以 RGB（红、绿、蓝）格式表示的颜色，即红色、绿色和蓝色各自具有独立的强度级别。混合这三种成分可以得到任何可见的颜色色调。绿色和红色混合会得到黄色，红色和蓝色混合会得到紫色，等等。

每个成分分配 1 个字节，也就是说，它可以取 0 到 255 之间的值。例如，所有成分都为 0 时产生黑色，而三个成分都为最大值 255 时混合成白色。

如果我们用十六进制表示法将颜色表示为 uint 类型，那么颜色的分布如下：0x00BBGGRR，其中 RR、GG 和 BB 是单字节无符号整数。

为了方便用户，MQL5 支持一种特殊形式的字面量来记录颜色常量。字面量表示由逗号分隔并括在单引号中的三个数字组成的三元组。在字面量前面放置字符 C。例如，C'0,128,255' 表示红色成分值为 0、绿色成分值为 128、蓝色成分值为 255 的颜色。也可以使用数字的十六进制表示法：C'0x00,0x80,0xFF'。

此外，MQL5 中嵌入了一长串预定义的颜色色调，所有这些颜色都以 clr 开头。例如，clrMagenta（品红色）、clrLightCyan（浅青色）和 clrYellow（黄色）。当然，它们也包括三原色：clrRed（红色）、clrGreen（绿色）和 clrBlue（蓝色）。完整的列表可以在 MetaEditor 帮助文档中找到。

下面是一些设置颜色的示例（也可以在文件 MQL5/Scripts/MQL5Book/p2/TypeColor.mq5 中找到）：

c
void OnStart()
{
  color y = clrYellow;         // 黄色（clrYellow）
  color m = C'255,0,255';      // 紫红色（clrFuchsia）
  color x = C'0x88,0x55,0x01'; // x = 136,85,1（没有这样的预定义颜色）
  color n = 0x808080;          // 灰色（clrGray）
}

枚举 ​

枚举是 MQL5 中内置的一组类型，每种枚举类型都包含一组用于描述相关概念或属性的命名常量。这些常量也被称为枚举元素。

例如，枚举 ENUM_DAY_OF_WEEK 包含了一周中所有日期对应的常量：

标识符（ID）描述值
SUNDAY星期日0
MONDAY星期一1
TUESDAY星期二2
WEDNESDAY星期三3
THURSDAY星期四4
FRIDAY星期五5
SATURDAY星期六6

枚举 ENUM_ORDER_TYPE 描述了 MetaTrader 5 中支持的所有订单类型：

标识符（ID）描述值
ORDER_TYPE_BUY市价买入订单0
ORDER_TYPE_SELL市价卖出订单1
ORDER_TYPE_BUY_LIMIT限价买入挂单2
ORDER_TYPE_SELL_LIMIT限价卖出挂单3
ORDER_TYPE_BUY_STOP止损买入挂单4
ORDER_TYPE_SELL_STOP止损卖出挂单5
ORDER_TYPE_BUY_STOP_LIMIT当达到订单价格时，在 StopLimit 价格处下达限价买入挂单6
ORDER_TYPE_SELL_STOP_LIMIT当达到订单价格时，在 StopLimit 价格处下达限价卖出挂单7
ORDER_TYPE_CLOSE_BY通过反向订单平仓的订单8

MQL5 中有几十种不同的枚举类型。它们的名称都以 “ENUM_” 作为前缀。我们将在学习相关领域知识的过程中逐步了解它们。

每个枚举类型都是一个独立的类型。然而，它们的内部表示是相同的，即四字节整数（int）。每个枚举常量都用一个或另一个数字进行编码，但在大多数情况下，程序员不需要记住这些数字，因为使用枚举的意义就在于用清晰明了的标识符来替代内部表示。

编译器确保枚举值始终是已定义的常量之一。否则，将会出现警告或编译错误（根据具体情况，参见示例）。

下面是 ENUM_DAY_OF_WEEK 枚举在代码中的实际使用示例（脚本 MQL5/Scripts/MQL5Book/p2/TypeEnum.mq5）：

c
void OnStart()
{
  ENUM_DAY_OF_WEEK sun = SUNDAY;     // sun = 0
  ENUM_DAY_OF_WEEK mon = MONDAY;     // mon = 1
  ENUM_DAY_OF_WEEK tue = TUESDAY;    // tue = 2
  ENUM_DAY_OF_WEEK wed = WEDNESDAY;  // wed = 3
  ENUM_DAY_OF_WEEK thu = THURSDAY;   // thu = 4
  ENUM_DAY_OF_WEEK fri = FRIDAY;     // fri = 5
  ENUM_DAY_OF_WEEK sat = SATURDAY;   // sat = 6
  
  int i = 0;
  ENUM_DAY_OF_WEEK x = i; // 警告：隐式枚举转换
  ENUM_DAY_OF_WEEK y = 1; // 可以，等于 MONDAY
  ENUM_ORDER_TYPE buy = ORDER_TYPE_BUY;   // buy = 0
  ENUM_ORDER_TYPE sell = ORDER_TYPE_SELL; // sell = 1
  // ...
  
  // 警告：隐式转换
  //          从 'enum ENUM_DAY_OF_WEEK' 转换为 'enum ENUM_ORDER_TYPE'
  //          'ENUM_ORDER_TYPE::ORDER_TYPE_SELL' 将被使用
  //          而不是 'ENUM_DAY_OF_WEEK::MONDAY'
  ENUM_ORDER_TYPE type = MONDAY;
  // 编译错误：取消注释以重现
  // ENUM_DAY_OF_WEEK day = ORDER_TYPE_CLOSE_BY; // 无法转换枚举
  // ENUM_DAY_OF_WEEK z = 10; // '10' - 无法转换枚举
}

一周中所有日期的常量都用 0 到 6 的数字进行编码，以星期日为起始点。基本上，常量不一定必须是连续的数字，也不一定从 0 开始。有些枚举类型并非如此。

请注意，相同的常量在不同的枚举类型中可能表示不同的含义。例如，在 ENUM_ORDER_TYPE 枚举中的订单类型 ORDER_TYPE_BUY 和 ORDER_TYPE_SELL，其值（0 和 1）与 ENUM_DAY_OF_WEEK 枚举中星期日（SUNDAY）和星期一（MONDAY）的值是相同的。

当将简单整数变量 i 的值复制到枚举变量 x 中时，编译器会给出警告，因为在程序执行阶段，变量 i 中可能包含除了允许的常量之外的值。

在变量 y 中，我们记录了数字 1，它表示 MONDAY，编译器认为这是一个正确的操作。

尝试将一个枚举的常量写入另一个枚举的变量中（如上面示例中为变量 type 赋值 MONDAY），可能会引发关于隐式类型转换的警告。当被写入的常量与目标枚举的某个元素具有相同的值时，就会发生这种情况。换句话说，两个枚举中各自有一个元素具有相关的值。然后编译器会自动替程序员进行隐式转换，但它会通过警告来“提醒”程序员检查一切是否按预期进行：MONDAY 将被替换为 ORDER_TYPE_SELL 这一事实确实很奇怪；不过，我们在此处是为了举例说明而故意这样做的。

如果被复制的元素在值上与另一个枚举的任何元素都不匹配，就会生成编译错误，因为无法进行隐式转换，比如将 ORDER_TYPE_CLOSE_BY 写入变量 day 中。

带有变量 z 的注释字符串也会导致编译错误，因为值 10 不属于 ENUM_DAY_OF_WEEK。如果程序员确定在特殊情况下仍然需要在枚举类型变量中记录一个任意值，他们可以使用显式类型转换。

显式和隐式类型转换将在“类型转换”部分进行讨论。

MQL5 允许程序员使用关键字 enum 声明自己的应用枚举类型。这一特性将在下一部分“自定义枚举（enum）”中进行介绍。

自定义枚举 ​

自定义枚举在结构上基于 int 类型，并且使用它们的原则与上一部分讨论的内置枚举完全一致。因此，我们在这里介绍自定义枚举，尽管严格来说，它们并非内置枚举。

要在 MQL5 代码中描述自己的枚举类型，你将使用关键字 enum。最简单的描述形式如下：

c
enum name
{
  element1,
  element2,
  element3
};

这段描述在程序中注册了一个名为 name 的枚举类型，其元素用花括号括起来，并用逗号分隔（元素的数量仅受限于 int 类型的最大值，从实际任务的角度来看，这可以认为没有限制）。标识符 element1、element2 和 element3 随后可以在程序中它们被定义的上下文中使用：全局范围（即在所有函数之外）或在函数内部（请参阅“变量的上下文、可见性和生命周期”部分）。

请注意花括号后面的分号。这是必需的，因为枚举描述是一个独立的语句，并且在任何 MQL5 语句后面都必须放置分号。

默认情况下，标识符取常量值，从 0 开始，每个后续值比前一个值大 1。如果需要，程序员可以为每个元素定义一个特定的值，在标识符右侧的“=”后面指定。例如，上面的语句等同于下面这个：

c
enum name
{
  element1 = 0,
  element2 = 1,
  element3 = 2
};

只允许指定常量或编译器在编译阶段能够计算的表达式作为值（更多详细信息，请参阅下面的示例）。

如果没有为所有元素定义值，跳过的值将根据最近的已知值（前面的值）自动计算，即在前一个已知值的基础上加 1。例如：

c
enum name
{
  element1 = 1,
  element2,
  element3 = 10,
  element4,
  element5
};

在这里，前两个元素取值为 1 和 2（计算得出），而从第三个元素开始取值为 10（显式指定）、11 和 12（最后两个元素是基于 10 计算得出的）。

在脚本 TypeUserEnum.mq5 中，有一些描述自定义枚举的示例：

c
const int zero = 0; // 运行时的值在编译时未知
 
enum
{
  MILLION = 1000000
};
 
enum RISK
{
  // OFF      = zero, // 错误：需要常量表达式
  LOW      = -1,
  MODERATE = -2,
  HIGH     = -3,
};
 
enum INCOME
{
  LOW      = 1,
  MODERATE = 2,
  HIGH     = 3,
  ENORMOUS = MILLION,
};
 
void OnStart()
{
  enum INTERNAL
  {
    ON,
    OFF,
  };
 
  // int x = LOW; // 访问不明确，可能是以下之一
  int x = RISK::LOW;
  int y = INCOME::LOW;
}

枚举 INTERNAL 展示了在函数内部描述枚举的可能性，并且这样做限制了该类型的可见性/可用性区域，这对于避免名称冲突很有用。

枚举 RISK 表明元素可以被赋予负值。注释掉的元素 OFF 无法被描述，因为尝试用一个非常量表达式对其进行初始化：在这种情况下，指定了变量 zero，而编译器无法计算其值。

在枚举 INCOME 中，元素 ENORMOUS 成功地用上面定义的另一个枚举 MILLION 中的值进行了初始化。枚举在编译时创建，因此它们可用于初始化表达式中。

名为 MILLION 的枚举没有名称，这样的枚举被称为匿名枚举。它们的基本用途是声明常量。然而，对于常量，更常使用有名称的枚举，因为它们允许根据元素的含义对元素进行分组。

由于在示例中定义了两个枚举，并且它们都有名称相同的元素，所以在声明变量 x 时指定 LOW 标识符会导致“访问不明确”的编译错误，因为不清楚指的是哪个枚举的元素。请注意，标识符可能具有（在这种情况下确实具有）不同的值。

为了解决这个问题，有一个特殊的上下文运算符：两个冒号“::”。它们有助于形成语言元素（在我们的例子中是枚举元素）的完整标识符：首先指定枚举名称，然后是运算符“::”，之后是元素标识符。例如：RISK::LOW 和 INCOME::LOW。我们将在相关部分了解所有运算符。

空类型（Void 类型） ​

void 类型是一种特殊的类型。它表示空值（无类型），并且不占用任何内存。它仅用于描述不返回任何值或没有任何参数的函数。我们已经了解过这样一个函数的示例：在第 1 部分的 HelloChart 脚本中的 OnStart 函数。关于这一点，我们将在“函数”部分进行更详细的讨论。

不可能使用 void 类型来描述变量；然而，在描述对类的任意对象的引用时，它是基本类型。这一特性将在第 3 部分处理面向对象编程的内容中进行介绍。
